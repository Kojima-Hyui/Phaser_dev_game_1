# サイバーパンク・ローグライトシューター

## 1. 企画概要

Phaser3を使用して開発する、SF・サイバーパンクをテーマとした2Dローグライトシューティングゲーム。オープンエリア型のマップを探索し、遠距離武器で敵を倒しながら成長していく。幾何学形状を用いたミニマルなビジュアルデザインにより、システム実装に集中し、趣味プロジェクトとして楽しみながら開発する。

## 2. 背景・目的

### 2.1 背景

- Phaser3を使ったゲーム開発の技術習得を目指している
- ローグライトジャンルの魅力（ランダム性、リプレイ性、成長要素）を体験したい
- 個人で完結できる規模のプロジェクトを求めている

### 2.2 目的

- **趣味・個人プロジェクト**として楽しみながらゲーム開発を行う
- Phaser3の実践的なスキルを習得する
- ローグライトゲームのコアメカニクスを理解し実装する
- 完成したプレイ可能なゲームを作り上げる

## 3. 対象

### ターゲットプレイヤー

- ローグライトゲームが好きなプレイヤー
- シンプルなグラフィックでもゲーム性を重視するプレイヤー
- PC（ブラウザまたはデスクトップ）でプレイするユーザー

### 開発者

- 個人開発（あなた自身）

## 4. 現状の課題

- Phaser3でのゲーム開発経験が不足している
- ローグライトゲームの設計・実装ノウハウが必要
- 個人開発のため、リソース（時間・アセット）に制約がある
- スコープが大きくなりすぎると完成しないリスクがある

## 5. 解決策・実施内容

### 5.1 ゲームの基本仕様

**ジャンル**

- 2Dトップダウンビュー ローグライトシューター
- オープンエリア型の探索

**世界観・テーマ**

- SF・サイバーパンク
- 未来都市、テクノロジー、荒廃した世界

**ビジュアルスタイル**

- シンプルな幾何学形状（四角、円、三角形など）
- ミニマルデザインで開発効率を重視
- カラーパレットはサイバーパンク風（ネオン、暗い背景）

**操作方法**

- キーボード操作のみ
  - WASD: プレイヤー移動
  - 矢印キー or マウス: 照準・射撃方向
  - スペースキー: 射撃
  - その他: スキル発動、インベントリなど

### 5.2 コアゲームシステム

**戦闘システム**

- 遠距離攻撃（銃・レーザーなど）中心
- シンプルな弾道計算
- 敵AIは基本的な追跡・攻撃パターン
- 弾幕要素は控えめ（アクション性は高すぎない）

**成長システム（ハイブリッド型）**

_ラン中の成長_

- レベルアップによるステータス上昇
- ステージクリア時の武器・アイテム選択
- 一時的なパワーアップ

_永続的な強化（メタ進行）_

- 通貨（クレジット）の蓄積
- 新武器・スキルのアンロック
- 基礎ステータスの永続強化
- キャラクター（プレイアブル）の追加

**ダンジョン/マップ**

- オープンエリア型のランダム生成マップ
- 複数のエリアをつなぐ構造
- エリアごとに難易度が上昇
- ボスエリアの設置

**アイテム・武器システム**

- シンプルな設計（10-20種類程度）
  - 武器: 5-7種類（ピストル、ショットガン、レーザーライフル、スナイパーなど）
  - パッシブアイテム: 5-7種類（移動速度UP、攻撃力UP、体力UPなど）
  - アクティブスキル: 3-5種類（シールド、ダッシュ、範囲攻撃など）
- レアリティシステム（Common, Rare, Epic）
- シナジー要素（特定の組み合わせでボーナス）

### 5.3 開発フェーズ

**フェーズ1: プロトタイプ（コア実装）**

- 基本操作（移動、射撃）
- シンプルな敵AI（1-2種類）
- 当たり判定とダメージ処理
- 基本的なUI（HP、スコア）

**フェーズ2: ゲームループ確立**

- オープンエリア型マップの生成
- 複数の敵タイプ追加（3-5種類）
- レベルアップシステム
- ゲームオーバー・リスタート機能

**フェーズ3: 成長システム実装**

- 武器システム（複数武器の実装と切り替え）
- アイテムシステム
- 永続強化システム（メタ進行）
- 通貨・アンロック機能

**フェーズ4: コンテンツ拡充**

- 武器・アイテムの追加
- ボスエネミーの実装
- ステージバリエーション
- エフェクト・サウンド追加

**フェーズ5: 調整・完成**

- バランス調整
- バグ修正
- UI/UX改善
- 最終ポリッシュ

### 5.4 技術スタック

- **ゲームエンジン**: Phaser3 (最新安定版)
- **言語**: TypeScript（型安全性を確保）
- **ビルドツール**: Vite または Webpack
- **エディタ**: VS Code + 関連拡張機能
- **バージョン管理**: Git
- **アセット**:
  - グラフィック: Canvas API / 幾何学形状のプログラム生成
  - サウンド: フリー素材（効果音・BGM）または後回し

## 10. リスクと対策

| リスク                             | 影響度 | 対策                                                        |
| ---------------------------------- | ------ | ----------------------------------------------------------- |
| スコープクリープ（機能追加しすぎ） | 高     | MVP（最小限の製品）を明確にし、追加機能はリスト化して後回し |
| 技術的な壁にぶつかる               | 中     | Phaser3コミュニティ、ドキュメント、チュートリアルを活用     |
| モチベーション低下                 | 中     | 小さな目標を設定し、進捗を可視化。完璧を求めすぎない        |
| パフォーマンス問題                 | 中     | 早めにプロファイリング、最適化手法の学習                    |
| アセット不足                       | 低     | 幾何学形状メインなので影響小。フリー素材で補完              |
| 完成しない                         | 高     | 「動くバージョン」を常に維持。段階的リリースを検討          |

### リスク軽減のための原則

1. **Small Scope, Big Fun**: 機能は少なく、楽しさは大きく
2. **Iterate Early**: 早めにプレイ可能な状態を作り、反復改善
3. **Feature Freeze**: ある時点で機能追加を止め、完成に集中
4. **Done is Better than Perfect**: 完璧より完成を優先
