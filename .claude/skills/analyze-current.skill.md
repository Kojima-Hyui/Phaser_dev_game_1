このプロジェクトの現行分析を実施します。以下の手順で進めてください：

## 分析の実施手順

1. **既存の分析ファイルの確認**
   - docsディレクトリ内の「現行分析_*.md」ファイルを確認
   - 最新の採番を特定（存在しない場合は1から開始）

2. **包括的な分析の実行**
   以下の観点から、複数のExploreエージェントを並列で起動して分析を行ってください：

   a) **プロジェクト構造分析** (Explore agent)
      - ディレクトリ構造とファイル構成
      - 主要な技術スタック
      - 依存関係とパッケージ構成

   b) **アーキテクチャ分析** (Explore agent)
      - コードの設計パターン
      - モジュール間の関係性
      - データフローとステート管理

   c) **機能分析** (Explore agent)
      - 実装されている主要機能
      - ユーザーインタラクション
      - ビジネスロジックの概要

   d) **技術的課題・改善点の分析** (Explore agent)
      - コード品質の評価
      - パフォーマンスの問題
      - セキュリティの懸念事項
      - 技術的負債

3. **分析結果の統合と日本語レポート作成**
   各エージェントからの分析結果を統合し、以下の構造で日本語レポートを作成：

   ```markdown
   # 現行分析レポート

   **分析日時**: YYYY年MM月DD日
   **プロジェクト**: [プロジェクト名]

   ## 1. エグゼクティブサマリー
   - プロジェクトの概要
   - 主要な発見事項
   - 重要な推奨事項

   ## 2. プロジェクト構造
   - ディレクトリ構成
   - 技術スタック
   - 依存関係

   ## 3. アーキテクチャ
   - 設計パターン
   - モジュール構成
   - データフロー

   ## 4. 機能概要
   - 実装済み機能一覧
   - ユーザーインタラクション
   - ビジネスロジック

   ## 5. 技術的評価
   - コード品質
   - パフォーマンス
   - セキュリティ

   ## 6. 課題と改善提案
   - 現在の課題
   - 短期的改善提案
   - 長期的改善提案

   ## 7. 結論
   - 総合評価
   - 次のステップ
   ```

4. **ファイルの保存**
   - ファイル名: `現行分析_YYYYMMDD_N.md` (Nはその日の採番)
   - 保存先: `docs/` ディレクトリ
   - 既に同じ日付のファイルが存在する場合は、採番を増やす

5. **完了報告**
   - 作成したファイルのパスを表示
   - 分析の主要な発見事項を簡潔に要約

## 注意事項
- 分析は徹底的に行い、見落としがないようにしてください
- 複数のExploreエージェントを並列実行して効率化してください
- レポートは読みやすく、実用的な内容にしてください
- すべて日本語で記述してください
